Token efficiency · LongMemEval

Answering from memory costs 66% fewer input tokens.

On the public LongMemEval benchmark, a memory-less agent re-sends the whole conversation to answer. Korely answers from a compact get_context() block. Measured, deterministic, no LLM and no API key required to verify.

This measures token efficiency against a naive baseline that re-sends the full history every turn. It does not measure answer accuracy (a separate, judge-dependent metric, out of scope here).

Every question

Korely stays in a flat band; full history climbs.

Each green dot is one of the 178 questions: full-history input tokens (x) vs Korely input tokens (y). The red diagonal is break-even (y = x). On these oracle histories (up to ~17k tokens), Korely stays in a low band near its ~2k budget while the full-history cost climbs the diagonal.

What the block keeps

A compression, not a copy.

Evidence is present, not complete

For every question, at least one gold answer-evidence turn is kept in the block (a floor). On average it keeps 47% of the evidence turns: the block compresses the history, that is the point. This is not a recall@k score, and it is not an accuracy claim.

Accuracy is measured separately

Whether the compressed block is enough to answer correctly is the accuracy question. It needs a neutral LLM judge of a different model family than the reader, and is tracked separately, not claimed on this page.

Method, honestly

Stated assumptions, no cherry-picking.

Naive baseline

"full history" re-sends every turn on every question. Real agents window, truncate, or cache, so 66% is "vs an agent that resends everything", not vs every alternative.

Conservative split

The oracle split has no distractor sessions, so the full history is as small as it gets. The reduction is a lower bound; on the full long-context split it is larger.

Tokenizer & budget

Counts use tiktoken o200k_base for both sides (ratio apples-to-apples; not measured against the Llama reader's tokenizer, so treat as ±a few points). The ~2k budget is a soft target: 65 of 178 blocks exceed it.

Question mix

knowledge-update is a full 78/78 census; the other axes are 20-question subsamples and multi-session skews large. The overall number is robust: pooled 66%, median 62%, re-weighted 60%.
